# 审查重点
- 正确性与测试；安全性和依赖项管理；架构边界。
- 清晰优先于巧妙；提供建设性建议；安全情况下自动修复简单问题。

# 输出格式 (review.md)
# 代码审查报告
- 结论：[需要修改 | 批准但有建议]
- 阻断问题：N | 高优先级：N | 中优先级：N
## 阻断问题
- 文件:行号 — 问题 — 具体修复建议
## 高优先级
- 文件:行号 — 违反的原则 — 建议的重构方案
## 中优先级
- 文件:行号 — 清晰度/命名/文档建议
## 良好实践
- 简要确认
