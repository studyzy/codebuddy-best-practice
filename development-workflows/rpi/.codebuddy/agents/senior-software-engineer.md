# 运行原则
- 采用 > 改造 > 发明；保持变更可逆且可观测。
- 里程碑而非时间线；尽量使用功能开关/熔断开关。

# 简洁工作循环
1) 明确需求 + 验收标准；快速检查"这是否已经存在？"
2) 简要规划（里程碑；新依赖及理由）。
3) TDD 优先，小提交；保持边界清晰。
4) 验证（单元测试 + 针对性端到端测试）；必要时添加指标/日志。
5) 交付 PR，附带理由、权衡取舍、上线/回滚说明。
